About Glowbloom

Glowbloom is a brain-training game suite by TreeAnts Technologies. It runs as a responsive single-screen web app, wrapped in a native Android/iOS shell (Flutter WebView) with native Google sign-in and Google AdMob. Phase 1 ships a memory game; three more brain games are planned, each targeting a different cognitive skill.

Games

GameSkillStatusWhat it does
Glowbloom MemorySequence memoryLiveWatch the glowbuds light up, then tap them back in order. Levels grow; Buds = lives, Light = score, Radiance = streak multiplier. Buy-a-Bud, 10s input timer, Sargam/Beeps/Mute sounds.
BloomstormSpeed & focusPlannedTap blooming flowers fast, resist wilting decoys, build combos. Trains processing speed and impulse control.
Lumen TrailsSpatial memoryPlannedA glowing path flashes across a grid; retrace it from memory while avoiding hazards.
SpectraLogic & deductionPlannedArrange coloured light orbs to satisfy clue constraints — a calmer, puzzle-style challenge.

Player features

  • Registration & login — Google (native on app, popup on web), Email OTP (Microsoft Graph), Mobile (optional). Email is mandatory and verified; first/last name, birth-year and device/IP captured.
  • Profile — upload a photo, view name/email, and share your stats & rank via the native share sheet.
  • Feedback — players send a category (Idea/Bug/Other) + 1–5 star rating + message, stored in the database.
  • Leaderboards — Today / Week / Month / Overall windows (IST), ties share a rank, crown for #1.
  • Home dashboard — games played, total Light, average and best level & score.
  • Sessions — stay signed in for 24 hours or until logout. How to play available from login and home.
  • Design — single-screen, no-scroll layout on every device; TreeAnts vibrant theme; 3-second branded splash.

Admin module (this panel)

  • Dashboard — KPIs (users, new, active DAU/WAU/MAU), 30-day sign-up/login/games trends, sign-up-method & platform breakdowns, top players, feedback summary.
  • Email config — choose provider (Microsoft Graph primary, Amazon SES optional), set the sender mailbox and credentials; applies live, no restart.
  • Test email — send a branded test and see the exact provider error on failure.
  • Overview — recent registrations and auth events with device/platform/IP.
  • Messages — edit any user-facing alert text; new alerts appear here automatically; blank = reset to default.
  • Game — tune difficulty & scoring live (pattern growth, recall time, starting/buyable Buds, bud cost, Light per tile, round bonus); every change is logged to a DB history.
  • Feedback — read everything players have submitted.
  • About — this page.

Monetization

Google AdMob in the app: a banner on every screen plus a rewarded ad (“watch for a free Bud”). Currently using Google test ad IDs — replace with real IDs and add app-ads.txt before publishing. Revenue and Play/App Store install numbers require connecting AdMob, Google Play Console and App Store Connect APIs (shown as “Not connected” on the Dashboard until then).

Architecture & data

Node/Express + Prisma + PostgreSQL backend on AWS EC2 (nginx + SSL + PM2). Flutter WebView wrapper for Android/iOS with native Google sign-in and AdMob bridges. Stored data: user profile, auth logs (device/browser/IP/platform), scores, light totals, feedback, and game-config change history.
